   ### ఫస్ట్ కాంట్రిబ్యూషన్స్ (First Contributions)

ఈ ప్రాజెక్ట్ ప్రారంభదశలో ఉన్నవారికి వారి మొదటి కాంట్రిబ్యూషన్ సులభంగా చేసుకునే మార్గాన్ని చూపించడానికి మరియు గైడ్ చేయడానికి రూపొందించబడింది. మీరు మీ మొదటి కాంట్రిబ్యూషన్ చేయాలనుకుంటున్నట్లయితే, దిగువ ఇచ్చిన స్టెప్స్ ను అనుసరించండి.

మీకు క‌మాండ్ లైన్ వాడటంలో సౌకర్యం లేకపోతే, GUI టూల్స్ ఉపయోగించి ట్యుటోరియల్స్ ఇక్కడ ఉన్నాయి.

మీ కంప్యూటర్లో git లేకుంటే, దయచేసి ఇన్స్టాల్ చేయండి

### రిపోజిటరీని ఫోర్క్ చేయండి

ఈ రిపోజిటరీని ఫోర్క్ చేయడానికి పేజీ పైభాగంలో ఉన్న Fork బటన్ పై క్లిక్ చేయండి.
దీనివల్ల ఈ రిపోజిటరీ కాపీ మీ GitHub అకౌంట్లో తయారవుతుంది.

### రిపోజిటరీని క్లోన్ చేయండి

<img align="right" width="300" src="https://firstcontributions.github.io/assets/Readme/clone.png" alt="రిపోజిటరీని క్లోన్ చేయండి" />

ఇప్పుడు ఫోర్క్ చేసిన రిపోజిటరీని మీ కంప్యూటర్లో క్లోన్ చేయండి.

మీ GitHub అకౌంట్లో ఫోర్క్ చేసిన రిపోజిటరీని తెరవండి.

Code బటన్ పై క్లిక్ చేయండి → SSH టాబ్ ఎంచుకోండి → copy url to clipboard ఐకాన్ పై క్లిక్ చేయండి.

తరువాత టెర్మినల్ తెరిచి ఈ కమాండ్ నడపండి:
```bash
git clone "మీరు కాపీ చేసిన URL"
``` 
ఇక్కడ "url you just copied" (ఉల్లేఖిత చిహ్నాలు లేకుండా) అనేది ఈ రిపోజిటరీకు URL, అంటే మీరు ఫోర్క్ చేసిన ప్రాజెక్ట్ URL. URL పొందడానికి మునుపటి దశలను చూడండి.

<img align="right" width="300" src="https://firstcontributions.github.io/assets/Readme/copy-to-clipboard.png" alt="URL ను కాపీ చేయండి" />

ఉదాహరణ:

```bash
git clone git@github.com:this-is-you/first-contributions.git
```

this-is-you స్థానంలో మీ GitHub యూజర్ నేమ్ రాయండి.
ఇది GitHub నుండి ప్రాజెక్ట్ కంటెంట్ ను మీ కంప్యూటర్లో కాపీ చేస్తుంది.

### బ్రాంచ్ సృష్టించండి

మీ కంప్యూటర్లో రిపోజిటరీ డైరెక్టరీకి వెళ్ళండి:

```bash
cd first-contributions
```
తరువాత కొత్త బ్రాంచ్ సృష్టించండి:

```bash
git switch -c your-new-branch-name
```
ఉదాహరణ:

```bash
git switch -c add-alonzo-church
```

<details> 
<summary> <strong>git switch లో ఎర్రర్లు వస్తే:</strong> </summary>

పాత వెర్షన్ Git ఉపయోగిస్తే, ఈ కమాండ్ ఉపయోగించండి:

```bash
git checkout -b మీ-కొత్త-బ్రాంచ్-నేమ్
```

</details>

### అవసరమైన మార్పులు చేయండి మరియు కమిట్ చేయండి

Contributors.md ఫైల్ ఓపెన్ చేసి, మీ పేరు చేర్చండి.
ఫైల్ చివరలో లేదా మొదట్లో కాకుండా మధ్యలో ఎక్కడైనా చేర్చండి.
ఫైల్ సేవ్ చేయండి.

<img align="right" width="450" src="https://firstcontributions.github.io/assets/Readme/git-status.png" alt="గిట్ స్థితి" />

ప్రాజెక్ట్ డైరెక్టరీలో ఈ కమాండ్ నడిపితే, మార్పులు కనిపిస్తాయి:  git status
    
మార్పులను బ్రాంచ్‌లో జోడించడానికి:

```bash
git add Contributors.md
```

ఇప్పుడు ఆ మార్పులను `git commit` కమాండ్ ఉపయోగించి కమిట్ చేయండి:

```bash
git commit -m "Add your-name to Contributors list"
```

your-name స్థానంలో మీ పేరు ఉంచండి.

### GitHubకి పుష్ చేయండి

```bash
git push -u origin your-branch-name
```

మీరు ముందుగా సృష్టించిన బ్రాంచ్ పేరుతో your-branch-name ని మారుస్తూ ఉపయోగించండి.

<details> <summary> <strong>పుష్ చేసే సమయంలో ఏవైనా errors వస్తే, ఇక్కడ క్లిక్ చేయండి:</strong> </summary>

### సత్యాపన లో లోపం
<pre>remote: Password authenticationకి 2021 ఆగస్టు 13న మద్దతు తీసివేయబడింది. దయచేసి వ్యక్తిగత యాక్సెస్ టోకెన్ (Personal Access Token) ఉపయోగించండి. remote: మరిన్ని వివరాలకు చూడండి: https://github.blog/2020-12-15-token-authentication-requirements-for-git-operations/ fatal: 'https://github.com/&lt;your-username&gt;/first-contributions.git/'కి authentication విఫలమైంది</pre>

మీ ఖాతాకు SSH కీని సృష్టించడం మరియు కాన్ఫిగర్ చేయడం పై GitHub ట్యూటోరియల్ కి వెళ్లడానికి. (https://docs.github.com/en/authentication/connecting-to-github-with-ssh/adding-a-new-ssh-key-to-your-github-account)

అలాగే, మీ రిమోట్ అడ్రస్‌ను చెక్ చేయడానికి 'git remote -v' రన్ చేయవచ్చు.

ఇది ఇలా కనిపిస్తే:
  <pre>origin	https://github.com/your-username/your_repo.git (fetch)
  origin	https://github.com/your-username/your_repo.git (push)</pre>
  
ఈ కమాండ్ ఉపయోగించి మార్చండి:

  ```bash
  git remote set-url origin git@github.com:your-username/your_repo.git
  ```
  లేకపోతే, మీరు ఇంకా username మరియు password కోసం ప్రాంప్ట్ అవుతారు మరియు authentication error వస్తూనే ఉంటుంది.

</details>

## మీ మార్పులను సమీక్ష కోసం సమర్పించండి

మీ GitHub రిపోజిటరీకి వెళ్ళినప్పుడు, మీరు Compare & pull request బటన్‌ను చూడగలుగుతారు. ఆ బటన్‌పై క్లిక్ చేయండి.

<img style="float: right;" src="https://firstcontributions.github.io/assets/Readme/compare-and-pull.png" alt="పుల్ రిక్వెస్ట్ సృష్టించండి" />

తరువాత, పుల్ రిక్వెస్ట్‌ను సమర్పించండి.

<img style="float: right;" src="https://firstcontributions.github.io/assets/Readme/submit-pull-request.png" alt="పుల్ రిక్వెస్ట్ పంపండి" />

చిన్న కాలంలో, మీ మార్పులు ఈ ప్రాజెక్ట్ యొక్క ముఖ్య బ్రాంచ్లో మర్జ్ చేయబడతాయి. మార్పులు మర్జ్ అయిన తరువాత, మీరు నోటిఫికేషన్ ఇమెయిల్ పొందుతారు.

### ఇక్కడి నుండి ఎక్కడికి వెళ్లాలి?

అభినందనలు! మీరు కచ్చితమైన fork -> clone -> edit -> pull request వర్క్‌ఫ్లోని పూర్తి చేసారు, ఇది మీరు సాధారణంగా కాంట్రిబ్యూటర్‌గా ఎదుర్కొనే ప్రాసెస్.

మీ కాంట్రిబ్యూషన్‌ను ఉత్సవంగా జరుపుకోండి మరియు మీ స్నేహితులు మరియు ఫాలోవర్స్‌తో పంచుకోండి [web app](https://firstcontributions.github.io/#social-share).


మరిన్ని ప్రాక్టీస్ కావాలంటే, [code contributions](https://github.com/roshanjossey/code-contributions)
 చూడండి.

ఇప్పుడు, ఇతర ప్రాజెక్టులకు కాంట్రిబ్యూట్ చేయడం ప్రారంభిద్దాం. మీరు ప్రారంభించడానికి సులభమైన సమస్యలతో ప్రాజెక్టుల జాబితాను మేము సేకరించాము. [list of projects in the web app ](https://firstcontributions.github.io/#project-list).

 చూడండి.
