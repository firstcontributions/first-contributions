## మొదటి సహకారం (First Contribution)

  Git Bash వెర్షన్

మొదటిసారి ఏదైనా చేసినప్పుడు అది కష్టంగా అనిపిస్తుంది. ముఖ్యంగా సహకారంతో కూడిన (collaborative) పని చేస్తున్నప్పుడు తప్పులు జరుగుతాయేమో అని భయం ఉంటుంది. కానీ ఓపెన్ సోర్స్ అంటేనే సహకారం - కలిసి పనిచేయడం.

కొత్త ఓపెన్ సోర్స్ కంట్రిబ్యూటర్లకు నేర్చుకునే మార్గాన్ని సులభతరం చేయాలని మేము భావించాము. కేవలం ట్యుటోరియల్స్ చదవడం లేదా వీడియోలు చూడటం ద్వారా మాత్రమే కాకుండా - ప్రాక్టికల్స్ చేయడం ద్వారా నేర్చుకుంటే చాలా వేగంగా అర్థమవుతుంది. ఈ ప్రాజెక్ట్ యొక్క ఉద్దేశ్యం కొత్తవారికి మార్గనిర్దేశం చేయడం మరియు వారు తమ మొదటి సహకారం అందించే అనుభవాన్ని సులభతరం చేయడం.

గుర్తుంచుకోండి, మీరు ఎంత సౌకర్యవంతంగా భావిస్తే, మీరు అంత బాగా నేర్చుకుంటారు.

మీరు విండోస్ ఉపయోగిస్తుంటే మరియు మీ వద్ద Git Bash లేకపోతే, ఇక్కడి నుండి ఇన్‌స్టాల్ చేసుకోండి.

---

### ఈ రిపోజిటరీని ఫోర్క్ (Fork) చేయండి

ఈ పేజీకి కుడి వైపు పైభాగంలో ఉన్న **Fork** బటన్‌పై క్లిక్ చేయండి.
దీని ద్వారా మీ గిట్‌హబ్ (GitHub) ఖాతాలో ఈ రిపోజిటరీకి సంబంధించిన ఒక కాపీ క్రియేట్ అవుతుంది.

### రిపోజిటరీని క్లోన్ (Clone) చేయండి

ఇప్పుడు మీరు ఫోర్క్ చేసిన రిపోజిటరీని మీ కంప్యూటర్‌లో క్లోన్ చేయండి.

> ⚠️ **గమనిక:** అసలు (original) రిపోజిటరీని క్లోన్ చేయకండి, మీ స్వంత ఫోర్క్ నుండి మాత్రమే క్లోన్ చేయాలి.

**1.** మీ రిపోజిటరీ పేజీకి వెళ్లి, "Code" బటన్‌పై క్లిక్ చేసి, అక్కడ ఉన్న లింక్‌ను కాపీ చేయండి.

**2.** Git Bash అప్లికేషన్‌ను తెరవండి. విండోస్‌లో ఇది కింది చిత్రంలో చూపిన విధంగా కనిపిస్తుంది:

**3.** మీరు ప్రాజెక్ట్‌ను ఉంచాలనుకుంటున్న ఫోల్డర్‌కు వెళ్లండి:

```bash
cd <folder>

```

**4.** ఇప్పుడు రిపోజిటరీని క్లోన్ చేయండి:

```bash
git clone <repo-url>

```

**5.** క్లోన్ చేసిన డైరెక్టరీకి వెళ్లి, దానిని Visual Studio Code లో తెరవండి:

---

### ఒక కొత్త బ్రాంచ్ (Branch) ని క్రియేట్ చేయండి

కింది కమాండ్‌ను ఉపయోగించి కొత్త బ్రాంచ్‌ని క్రియేట్ చేసి, ఆ బ్రాంచ్‌కి మారండి:

```bash
git checkout -b <branch-name>

```

👉 మీ బ్రాంచ్‌కు `<add-your-name>` అని పేరు పెట్టండి. ఉదాహరణకు:

```bash
add-james-smith

```

---

### మార్పులు చేయండి మరియు కమిట్ (Commit) చేయండి

**1.** `Contributors.md` ఫైల్‌ను తెరవండి.
**2.** కిందికి స్క్రోల్ చేసి, అక్కడ మీ పేరును జోడించండి.
**3.** ఫైల్‌ను సేవ్ చేయండి.

**ఉదాహరణ:**

```markdown
[James Smith](https://github.com/jamessmith)

```

మార్పులు జరిగాయో లేదో చూడటానికి:

```bash
git status

```

ఫైల్‌ను స్టేజ్ (Stage) చేయడానికి:

```bash
git add file-name

```

కమిట్ చేయడానికి:

```bash
git commit -m "Add your-name to Contributors list"

```

👉 `<your-name>` ఉన్న చోట మీ పేరు రాయండి.

కమిట్‌ను చెక్ చేయాలనుకుంటే:

```bash
git log --oneline

```

---

### మార్పులను GitHub కు Push చేయండి

అన్ని దశలు పూర్తయిన తర్వాత, మార్పులను GitHub కి పంపాలి:

```bash
git push origin <branch-name>

```

---

### పుల్ రిక్వెస్ట్ (Pull Request) పంపండి

GitHub లో మీ రిపోజిటరీకి వెళితే **Compare & pull request** బటన్ కనిపిస్తుంది. దానిపై క్లిక్ చేయండి.

ఆ తర్వాత Pull Request ను సబ్‌మిట్ చేయండి.

త్వరలోనే మీ మార్పులు అసలు రిపోజిటరీలోని `master` బ్రాంచ్‌లో విలీనం (merge) అవుతాయి. మార్పులు విలీనం కాగానే మీకు ఈమెయిల్‌లో నోటిఫికేషన్ వస్తుంది.

---

### తర్వాత ఏమి చేయాలి?

అభినందనలు 🎉 మీరు ఇప్పుడే సాధారణ **fork → clone → edit → PR** వర్క్‌ఫ్లోను పూర్తి చేసారు, ఒక కంట్రిబ్యూటర్‌గా దీనిని మీరు పదే పదే ఉపయోగించాల్సి ఉంటుంది.

* 👉 మీ సహకారాన్ని సెలబ్రేట్ చేసుకోండి మరియు వెబ్ యాప్ నుండి మీ స్నేహితులతో పంచుకోండి.
* 👉 మీకు ఏవైనా ప్రశ్నలు ఉన్నా లేదా సహాయం కావాలన్నా మా Slack టీమ్‌లో చేరవచ్చు: స్లాక్ టీమ్‌లో చేరండి.
