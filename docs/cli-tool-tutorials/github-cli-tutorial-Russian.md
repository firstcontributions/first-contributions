# Первый вклад

| Git Bash | Версия Git Bash |
|----------|----------------|

Это сложно. Когда вы делаете что-то впервые — это всегда сложно. Особенно если вы сотрудничаете, ошибки кажутся чем-то пугающим. Но открытый исходный код — это про сотрудничество и совместную работу. Мы хотели упростить процесс обучения и сделать первый вклад в open source легче для новичков.

Чтение статей и просмотр туториалов может помочь, но что может быть лучше, чем попробовать самому? Цель этого проекта — дать руководство и упростить процесс первого вклада для новичков. Помните: чем комфортнее вы себя чувствуете, тем лучше вы учитесь. Если вы хотите сделать свой первый вклад — просто следуйте шагам ниже. Обещаем, будет интересно.

Если у вас нет Git Bash на Windows, [установите его](https://git-scm.com/download/win).

---

## Сделайте форк репозитория

Нажмите кнопку **Fork** в правом верхнем углу этой страницы.  
Это создаст копию репозитория в вашем аккаунте.

---

## Клонируйте репозиторий

Теперь клонируйте этот репозиторий на свой компьютер.

⚠️ Важно: не клонируйте оригинальный репозиторий. Перейдите в свой форк и клонируйте его.

Нажмите **Code** и скопируйте ссылку.

Откройте Git Bash.

Перейдите в папку, куда хотите сохранить проект:

```
cd <папка>
```

Клонируйте репозиторий:

```
git clone <repo-url>
```

Перейдите в папку проекта:

```
cd <repo-name>
```

---

## Создайте ветку

Создайте новую ветку с помощью команды:

```
git checkout -b <название-ветки>
```

Назовите её `<add-your-name>`, например:

```
add-ivan-ivanov
```

---

## Внесите изменения и закоммитьте их

Откройте файл `Contributors.md`, прокрутите вниз и добавьте своё имя, затем сохраните файл.

Пример:

```
[Ivan Ivanov](https://github.com/your-profile)
```

Проверьте изменения:

```
git status
```

Добавьте изменения в staging:

```
git add file-name
```

Сделайте коммит:

```
git commit -m "Add your-name to Contributors list"
```

Замените `<your-name>` на своё имя.

Проверьте коммит:

```
git log --oneline
```

---

## Отправьте изменения на GitHub

После выполнения всех шагов выполните:

```
git push origin <название-ветки>
```

---

## Отправьте изменения на проверку

Перейдите в свой репозиторий на GitHub — вы увидите кнопку  
**Compare & pull request**. Нажмите её.

Создайте pull request.
