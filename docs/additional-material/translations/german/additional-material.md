# Zusätzliche Informationen

Wir gehen davon aus, dass Sie das grundlegende Tutorial bereits abgeschlossen haben, bevor Sie hierher gekommen sind. Dieses Dokument enthält einige zusätzliche Informationen zu fortgeschrittenen Git-Techniken.

### [Ändern eines Commits](amending-a-commit.md)
Dieses Dokument enthält Informationen darüber, wie Sie einen Commit im Remote-Repository ändern können. Das Ändern eines Commits ist eine Möglichkeit, den letzten Commit zu ändern, den Sie in Ihrem aktuellen Branch vorgenommen haben. Dies kann hilfreich sein, wenn Sie die Commit-Nachricht bearbeiten müssen oder wenn Sie vergessen haben, Änderungen in den Commit aufzunehmen. Sie können einen Commit so lange ändern, bis Sie ihn in das Remote-Repository übertragen.
> Verwenden Sie diese Funktion, wenn Sie einen von Ihnen erstellten Commit anpassen müssen.

### [Synchronisieren Ihres Forks mit dem Repository](keeping-your-fork-synced-with-this-repository.md)
Dieses Dokument enthält Informationen darüber, wie Sie Ihr geforktes Repository mit dem Basis-Repository synchronisieren können. Dies ist wichtig, da Sie und hoffentlich viele andere zu diesem Projekt beitragen werden.
> Befolgen Sie diese Schritte, wenn Ihr Fork keine Änderungen im übergeordneten Repository aufweist.
