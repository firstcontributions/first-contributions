## ನಿಮ್ಮ Repository ಯಿಂದ Branch ಅನ್ನು ತೆಗೆಯುವದು

ನಿಂಗ ಈ ಟ್ಯುಟೋರಿಯಲ್ ಅನ್ನು ಇದುವರೆಗೆ ಅನುಸರಿಸಿದ್ದರೆ, ನಿಮ್ಮ Branch `<add-seu-nome>` ತನ್ನ ಗುರಿಯನ್ನು ಪೂರ್ಣಗೊಳಿಸಿದೆ, ಮತ್ತು ಅದನ್ನು ನಿಮ್ಮ ಸ್ಥಳೀಯ Repository ಯಿಂದ ಅಳಿಸುವ ಸಮಯ ಆಗಿದೆ. ಇದು ಅಗತ್ಯವಿಲ್ಲ, ಆದರೆ ಆ Branch ನ ಹೆಸರೇ ಅದರ ಉದ್ದೇಶವನ್ನು ಸ್ಪಷ್ಟಪಡಿಸುವ. ಅದರ ಅವಧಿ ಈ ನಿರ್ದಿಷ್ಟತೆಯಿಂದ ಸ್ವಲ್ಪ ಕಾಲ ಮಾತ್ರ ಇರಲಕ್ಕು.

ಮೊದಲು, `<add-seu-nome>` Branch ಅನ್ನು ನಿಮ್ಮ ಮುಖ್ಯ Branch (master) ಗೆ ವಿಲೀನಗೊಳಿಸುವ, ಆದ್ದರಿಂದ ಅದಕ್ಕೆ ಹೋಗುವ:
```
git checkout master
```

`<add-seu-nome>` ಅನ್ನು master ಗೆ ವಿಲೀನಗೊಳಿಸಿ:
```
git merge <add-seu-nome> master
```

`<add-seu-nome>` ಅನ್ನು ನಿಮ್ಮ ಸ್ಥಳೀಯ Repository ಯಿಂದ ತೆಗೆಯಿ:
```
git branch -d <add-seu-nome>
```

ಈಗ ನಿಂಗ ನಿಮ್ಮ ಸ್ಥಳೀಯ `<add-seu-nome>` Branch ಅನ್ನು ಅಳಿಸಿದ್ದೀರ, ಮತ್ತು ಎಲ್ಲವೂ ಸ್ವಚ್ಛ ಮತ್ತು ಕ್ರಮಬದ್ಧವಾಗಿ ಉಂಡು.
ಈ ಹಂತದಲ್ಲಿ, ನಿಂಗ ಇನ್ನೂ ನಿಮ್ಮ Fork ನಲ್ಲಿ `<add-seu-nome>` Branch ಅನ್ನು ಹೊಂದಿರಲಕ್ಕು. ಅದನ್ನು ಅಳಿಸುವ ಮೊದಲು, ನಿಂಗ ಈ Branch ನಿಂದ ನಡ Repository ಗೆ Pull Request ಕಳುಹಿಸಿದ್ದೀರಾ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ. ಹೀಗಾಗಿ ನಾ ಇದನ್ನು ಆಗಲೇ ವಿಲೀನಗೊಳಿಸಿಲ್ಲದಿದ್ದರೆ, ಅದನ್ನು ಅಳಿಸಬೇಡಿ.

ಆದರೆ, ನಾ ಆಗಲೇ ನಿಮ್ಮ Branch ಅನ್ನು ವಿಲೀನಗೊಳಿಸಿದ್ದರೆ ಮತ್ತು ನಿಂಗ ರಿಮೋಟ್ Branch ಅನ್ನು ಅಳಿಸುವಕ್ ಬಯಸಿದರೆ, ಈ ಆಜ್ಞೆಯನ್ನು ಬಳಸಿ:
```
git push origin --delete <add-seu-nome>
```

ಇತ್ತೀಚಿನ Branch ಗಳ್ ನಿಂಗ ಹೇಗೆ ನಿರ್ವಹಿಸಲಕ್ಕು ಎಂಬುದನ್ನು ನಿಂಗ ಈಗ ತಿಳಿದಿದ್ದೀರ.
ಕಾಲಾಂತರದಲ್ಲಿ, ನಡ ಸಾರ್ವಜನಿಕ Repository ಲ್ಲಿ ಅನೇಕ Commit ಗಳ್ ಸೇರಿಕೊಳ್ಳುವ. ನಿಮ್ಮ ಲ್ಯಾಪ್‌ಟಾಪ್‌ನಲ್ಲಿನ ಮತ್ತು Fork ನಲ್ಲಿನ ಮುಖ್ಯ Branch ಗಳ್ ಇನ್ನಷ್ಟು ಅಪ್‌ಡೇಟ್ ಆಗಿರುವುದಿಲ್ಲ. ಹೀಗಾಗಿ, ನಿಮ್ಮ Repository ಗಳ್ ನಡದರೊಂದಿಗೆ ಸಿಂಕ್ರೀನೈಸ್ ಮಾಡ್ವದಕ್ಕಾಗಿ, ಕೆಳಗಿನ ಹಂತಗಳ್ ಅನುಸರಿಸಿ.
