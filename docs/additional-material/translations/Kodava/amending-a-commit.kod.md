# Commit ಸರಿಪಡಿಸುವದು

ನಿಂಗ ನಿಮ್ಮ ರಿಮೋಟ್ Repository ಗೆ ಬದಲಾವಣೆಯನ್ನು Commit ಮಾಡಿದ ಪಿನ್ನೆ, ಅದರಲ್ಲಿ Commit ಸಂದೇಶದಲ್ಲಿ ಒಂದು ದೋಷ ಉಂಡು ಅಥವಾ ನಿಂಗ ನಿಮ್ಮ ಇತ್ತೀಚಿನ Commit ನಲ್ಲಿ ಒಂದು ಸಾಲಿನ ಕೋಡ್ ಸೇರಿಸುವಕ್ ಮರತಂಡಂಡು ಎಂಬುದು ನಿಮಗೆ ಪಿನ್ನೆ ತಿಳಿದರೆ ಏನ್ ಮಾಡುವದು?

ಈ ಪರಿಷ್ಕರಣೆಯನ್ನು ನಿಂಗ ಹೇಗೆ ಮಾಡುವದು? ಈ ಟ್ಯುಟೋರಿಯಲ್ ಅದನ್ನು ಆವರಿಸುವ.

## GitHub ಗೆ ಪುಶ್ ಮಾಡಿದ ಪಿನ್ನೆ ಇತ್ತೀಚಿನ Commit ಸಂದೇಶವನ್ನು ಪರಿವರ್ತಿಸುವದು

ಇದು ಫೈಲ್ ತೆರೆಯದೆ ಮಾಡುವ ವಿಧಾನ:
* ```git commit --amend -m "ನಿಮ್ಮ ಹೊಸ Commit ಸಂದೇಶ"``` ಎಂಬ ಆಜ್ಞೆಯನ್ನು ನಮೂದಿಸಿ.
* ```git push origin <Branch-ಹೆಸರು>``` ಆಜ್ಞೆಯನ್ನು ಕಾರ್ಯಗತಗೊಳಿಸಿ, ಬದಲಾವಣೆಗಳ್ Repository ಗೆ Commit ಮಾಡಲು.

ಟಿಪ್ಪಣಿ: ನಿಂಗ ```git commit --amend``` ಮಾತ್ರ ನಮೂದಿಸಿದರೆ, ನಿಮ್ಮ ಟೆಕ್ಸ್ಟ್ ಎಡಿಟರ್ ತೆರೆಯುವ ಮತ್ತು ನಿಂಗ Commit ಸಂದೇಶವನ್ನು ಸಂಪಾದಿಸುವಕ್ ಅವಕಾಶ ಸಿಕ್ಕುವ.
```-m``` ಫ್ಲ್ಯಾಗ್ ಅನ್ನು ಬಳಸುವುದರಿಂದ ಅದು ತಪ್ಪಿಸಲಾಗುವ.

## ಒಂದೇ ಒಂದು Commit ನಲ್ಲಿ ಬದಲಾವಣೆಗಳ್ ಮಾಡುವದು

ನಾಂಗ ಒಂದೇ ಒಂದು ಶಬ್ದವನ್ನು ಸೇರಿಸುವಂತ ಸಣ್ಣ ಬದಲಾವಣೆಯನ್ನು ಮಾಡಬೇಕಾದರೂ, ಆಗಲೇ ಅದನ್ನು ರಿಮೋಟ್ Repository ಗೆ ಪುಶ್ ಮಾಡಿದ್ದರೆ ಏನ್ ಮಾಡುವದು?

ಉದಾಹರಣೆಗೆ, ಇಲ್ಲಿ ನಡ Commit ಲಾಗ್ ಉಂಡು:
```
g56123f bot file ನಿರ್ಮಿಸಲಾಯಿತು
a2235d contributor.md ಅಪ್‌ಡೇಟ್ ಮಾಡಲಾಗಿದೆ
a5da0d bot file ಪರಿಷ್ಕರಿಸಲಾಯಿತು
```

ನಾ bot file ಗೆ ಒಂದು ಶಬ್ದ ಸೇರಿಸುವಕ್ ಮರತಂಡಂಡು ಎಂದು ಊಹಿಸುವ.

ಈ ಸಮಸ್ಯೆಯನ್ನು ಪರಿಹರಿಸುವಕ್ ಎರಡು ವಿಧಾನಗಳ್ ಉಂಡು. ಮೊದಲನೆಯದು ಹೊಸ Commit ಮಾಡುವದು:

```
g56123f bot file ನಿರ್ಮಿಸಲಾಯಿತು
a2235d contributor.md ಅಪ್‌ಡೇಟ್ ಮಾಡಲಾಗಿದೆ
a5da0d bot file ಪರಿಷ್ಕರಿಸಲಾಯಿತು
b0ca8f bot file ಗೆ ಶಬ್ದ ಸೇರಿಸಲಾಯಿತು
```

ಇದನ್ನು ಮಾಡುವ ಇನ್ನೊಂದು ವಿಧಾನವೆಂದರೆ, ಹಿಂದಿನ ```a5da0d``` Commit ಅನ್ನು ಪರಿಷ್ಕರಿಸಿ, ಹೊಸ ಶಬ್ದವನ್ನು ಸೇರಿಸಿ ಮತ್ತು GitHub ಗೆ ಒಂದೇ Commit ನಂತೆ ಪುಶ್ ಮಾಡುವದು.

ಇದು ಉತ್ತಮ ಆಯ್ಕೆಯಾಗಿ ಉಂಡು ಏಕೆಂದರೆ ಇದು ಕೇವಲ ಸಣ್ಣ ಬದಲಾವಣೆ.

### ಇದನ್ನು ಹೇಗೆ ಮಾಡುವದು?

* ಫೈಲ್ ಅನ್ನು ಪರಿಷ್ಕರಿಸಿ. ಈ ಸಂದರ್ಭದಲ್ಲಿ, ನಾ bot file ಗೆ ಹಿಂದಿನ ಶಬ್ದ ಸೇರಿಸುವಿ.
* ಪಿನ್ನೆ, ```git add <ಫೈಲ್-ಹೆಸರು>``` ಆಜ್ಞೆ ಬಳಸಿ ಸ್ಟೇಜಿಂಗ್ ಏರಿಯಾದಲ್ಲಿ ಸೇರಿಸಿ.

ಸಾಧಾರಣವಾಗಿ, ಫೈಲ್‌ಗಳ್ ಸ್ಟೇಜಿಂಗ್ ಏರಿಯಾದಲ್ಲಿ ಸೇರಿಸಿದ ಪಿನ್ನೆ, ನಾಂಗ ```git commit -m "ನಡ Commit ಸಂದೇಶ"``` ಬಳಸುವ, ಆದರೆ ಈಗ ನಾಂಗ ಹಿಂದಿನ Commit ಅನ್ನು ಪರಿಷ್ಕರಿಸುವಕ್ ಬಯಸುವ ಕಾರಣ, ಬದಲಿಗೆ ಈ ಆಜ್ಞೆಯನ್ನು ಬಳಸೋಕು:

* ```git commit --amend```
  ಇದು ನಿಮ್ಮ ಟೆಕ್ಸ್ಟ್ ಎಡಿಟರ್ ಅನ್ನು ತೆರೆದು, Commit ಸಂದೇಶವನ್ನು ಪರಿಷ್ಕರಿಸುವಕ್ ಅನುಮತಿಸುವ. ನಿಂಗ ಅದನ್ನು ಬದಲಾಯಿಸುವದು ಅಥವಾ ಅದೇ ರೀತಿ ಉಳಿಪದು.
* ಎಡಿಟರ್ ಅನ್ನು ಸೇವ್ ಮಾಡಿ ಮತ್ತು ಮುಚ್ಚಿ.
* ```git push origin <Branch-ಹೆಸರು>``` ಆಜ್ಞೆಯನ್ನು ಬಳಸಿ ಬದಲಾವಣೆಗಳ್ ಪುಶ್ ಮಾಡಿ.

ಈ ರೀತಿ, ಎರಡೂ ಬದಲಾವಣೆಗಳ್ ಒಂದೇ Commit ನಲ್ಲಿ ಉಳಿಯುವ.
